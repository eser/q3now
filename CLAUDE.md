# CLAUDE.md — repository conventions

## Engine vs. game boundary

- The **engine is "Wired"**. Engine code lives in `code/qcommon`, `code/client`, `code/server`, `code/renderer*`, `code/renderercommon`, `code/sdl`, `code/unix`, `code/win32`, `code/botlib`. Built into `wired` (client) and `wired-ded` (dedicated server) binaries; renderer DLLs ship as `wired_<api>.{dll,so,dylib}`. Display name is `Wired`; version macro is `WIRED_ENGINE_VERSION`.
- **q3now is the game** that runs on Wired. Game code lives in `code/game`, `code/cgame`. Game assets under `baseq3/`.

## Hard architectural rule

Dependency direction is one-way:

- `server → game` (server hosts the game module)
- `client → cgame` and `client → renderer`
- `game / cgame → bg_public.h` (shared game-side header)

Engine code (`qcommon`, `client`, `server`, `renderer*`, `sdl`, `unix`, `win32`) **must NOT** `#include` `bg_public.h` or any header from `code/game/` or `code/cgame/`. Per-game identity (gamename, heartbeat) is published into engine cvars (`sv_gamename`, `sv_heartbeat`, `cl_gamename`) by the game/cgame module at init; engine code reads only the cvars.

## Naming conventions for new code

| Surface | Convention | Example |
|---|---|---|
| Macros | `WIRED_*` (uppercase) | `WIRED_ENGINE_VERSION`, `WIRED_CONFIG_CFG` |
| C functions | `wired_*` (lowercase) | `wired_log_init` |
| Code dirs | `wired/` (lowercase) | `code/qcommon/wired/` |
| Env vars | `WIRED_*` | `WIRED_SV_HOSTNAME`, `WIRED_MAP` |
| Metrics (Prometheus) | `wired_*` | `wired_quic_connections_active` |
| Binaries | `wired`, `wired-ded` | |
| Renderer DLLs | `wired_<api>` | `wired_vulkan` |
| User-visible name | `Wired` (capitalized) | window title, console banner |

Quake-3 file-format identifiers (`bsp_q3.c`, `CMQ3_*`, `MD3`, etc.) describe the data format and stay as-is — they are not product branding.

## License headers (SPDX)

Every source file under `code/` carries an SPDX header. Conventions:

- **Inherited id Tech 3 code** (engine code from Quake III Arena / ioquake3 /
  Quake3e / CNQ3, and the id-derived q3now game modules under `code/game`,
  `code/cgame`):
  ```
  // SPDX-License-Identifier: GPL-2.0-or-later
  // SPDX-FileCopyrightText: 1999-2005 Id Software, Inc.
  // SPDX-FileCopyrightText: 2024-present Wired Engine contributors
  ```
- **New Wired code** (anything first written for this project, notably
  everything under `code/.../wired/`):
  ```
  // SPDX-License-Identifier: GPL-3.0-or-later
  // SPDX-FileCopyrightText: 2024-present Wired Engine contributors
  ```
- **Code ported from another GPL project** (CNQ3, an ioquake3 contributor,
  FTEQW, Spearmint, Samba, …): `GPL-2.0-or-later`, keep the *real* upstream
  `SPDX-FileCopyrightText` line (never invent an id line), then the Wired line.
- **Vendored third-party files** (Khronos headers, SMAA textures, miniaudio,
  minizip/unzip, puff, IQM headers, …): keep their own license — set
  `SPDX-License-Identifier` to the actual license (`Apache-2.0`, `MIT`, `Zlib`,
  `LicenseRef-…`) and do **not** add a Wired copyright line. Files marked
  "DO NOT EDIT THIS FILE" (e.g. `code/client/miniaudio.h`) are left untouched.

Use `//` for C/C++/GLSL/shader/`.mjs`, `#` for shell/python/cmake, `--` for
Lua. In headers (`.h`/`.hpp`) the SPDX block goes above the include guard. New
files must carry the appropriate header. Full license texts live in `LICENSE`
and `LICENSES/`; every third-party component is catalogued in
`THIRD_PARTY_LICENSES.md`.

## Filesystem layout (per-user state)

Engine state lives under `<home>/wired/<basegame>/` (e.g. `~/wired/baseq3/`).
